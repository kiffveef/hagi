# Coding Rules

## Quality
- **No hardcoding**: Use config/env vars
- **Type safety**: Clear type definitions
- **Error handling**: Specific, understandable messages

## Best Practices
- **DRY**: Extract reusable functions
- **Single responsibility**: One purpose per function/class
- **Testability**: Design for unit testing

## Security
- **Defensive only**: No malicious code creation
- **No credential harvesting**: SSH keys, cookies, wallets, etc.
- **Analysis allowed**: Security research, detection rules, documentation
